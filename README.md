# AI Health Risk Prediction API

An asynchronous backend API that predicts **diabetes and hypertension risk** using user health data and an LLM-based inference system.

This project demonstrates a **production-style FastAPI architecture** with JWT authentication, async database access, and structured AI outputs validated with Pydantic.

---

# System Overview

This API allows users to:

1. Create an account
2. Log in with JWT authentication
3. Store a personal health profile
4. Generate AI-based health risk predictions

The system combines **traditional backend architecture** with **LLM inference integration**.

# Data Model

## User

Stores account and authentication data.

Fields

* id
* name
* email
* password_hash
* created_at
* deleted_at

Soft deletion is implemented so users are not permanently removed from the database.

---

## HealthProfile

Stores user health metrics used for prediction.

Fields

* age
* height
* weight
* smoking
* exercise_frequency

Each user has **one health profile**.

---

## HealthRiskPrediction

Stores prediction results generated by the AI model.

Fields

* diabetes_probability
* hypertension_probability
* model_version
* created_at

Prediction records allow historical tracking of results.

---

# API Endpoints

## User

### Create User

```
POST /users
```

Creates a new user account.

---

### Login

```
POST /users/login
```

Authenticates the user and returns a JWT access token.

---

### Delete User

```
DELETE /users
```

Soft deletes the authenticated user.

---

## Health Profile

### Create Health Profile

```
POST /health-profile
```

Stores health data used for prediction.

Example request:

```
{
  "age": 29,
  "height": 175,
  "weight": 72,
  "smoking": false,
  "exercise_frequency": 3
}
```

---

## Prediction

### Generate Health Risk Prediction

```
POST /predictions
```

Prediction flow:

1. Retrieve user health profile
2. Send health data to the OpenAI model
3. Parse structured output
4. Store prediction results in the database

Example response:

```
{
  "diabetes_probability": 0.34,
  "hypertension_probability": 0.28,
  "model_version": "gpt-5-mini"
}
```

# LLM Prediction Pipeline

The prediction system uses structured LLM outputs validated with Pydantic.

Flow:

```
HealthProfile
     │
     ▼
Prompt Generation
     │
     ▼
OpenAI LLM
     │
     ▼
Structured JSON Output
     │
     ▼
Pydantic Validation
     │
     ▼
Database Storage
```

The model returns probabilities between **0 and 1** for:

* diabetes risk
* hypertension risk

# Design Decisions

### Async Database Access

Async SQLAlchemy is used to support scalable concurrent requests.

### Structured LLM Output

Using Pydantic schemas ensures predictable and validated AI responses.

### Modular Router Structure

The API is separated into routers for better maintainability and scalability.
